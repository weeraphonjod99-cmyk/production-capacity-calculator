# Production Capacity Planner

เว็บแอปสำหรับวางแผนกำลังการผลิตของเครื่องจักร สามารถเพิ่มเครื่อง เพิ่มงานผลิต วิเคราะห์โหลดงานรายเครื่อง และสรุปกำลังการผลิตรวมได้ทันที

## ลิงก์ออนไลน์

เปิดใช้งานผ่าน GitHub Pages:

https://weeraphonjod99-cmyk.github.io/production-capacity-calculator/

รายงานวิเคราะห์กำลังการผลิตรายเครื่อง:

https://weeraphonjod99-cmyk.github.io/production-capacity-calculator/analysis.html

## ความสามารถหลัก

- เพิ่ม แก้ไข และลบเครื่องจักร
- เพิ่ม แก้ไข และลบงานผลิต พร้อมผูกงานเข้ากับเครื่อง
- คำนวณกำลังผลิตต่อวันและต่อเดือน
- คำนวณโหลดงานเทียบกำลังผลิตของแต่ละเครื่อง
- แสดงสถานะ `ปกติ`, `โหลดสูง`, `เกินกำลัง`, และ `ยังไม่มีงาน`
- ตั้งค่าวันทำงานต่อเดือนและเปอร์เซ็นต์เตือนโหลดสูง
- นำเข้าไฟล์ `.xlsx`, `.xlsm`, `.csv`, `.tsv`
- ส่งออกข้อมูลเป็น CSV
- ติดตั้งเป็นแอป PWA ได้จาก browser
- ใช้งานออฟไลน์ได้หลังเปิดเว็บครั้งแรก
- บันทึกข้อมูลไว้ในเครื่องผู้ใช้ด้วย `localStorage`

## สูตรคำนวณ

```text
กำลังผลิตต่อวัน =
((ชั่วโมงทำงานต่อวัน x 60 - เวลาหยุดต่อวัน) x 60 / Cycle Time)
x ชิ้นต่อรอบ x (OEE / 100) x จำนวนเครื่อง
```

```text
กำลังผลิตต่อเดือน =
กำลังผลิตต่อวัน x วันทำงานต่อเดือน
```

```text
Utilization (%) =
ยอดงานผลิตต่อเดือน / กำลังผลิตต่อเดือน x 100
```

## รูปแบบข้อมูลนำเข้า

ไฟล์ Excel/CSV ควรมีหัวคอลัมน์ที่ตรงกับรายการด้านล่างบางส่วนหรือทั้งหมด:

### ข้อมูลเครื่อง

- `เครื่องจักร` หรือ `Machine`
- `ประเภท` หรือ `Type`
- `จำนวนเครื่อง` หรือ `Quantity`
- `ชั่วโมง/วัน` หรือ `Hours per day`
- `หยุด/วัน (นาที)` หรือ `Downtime minutes`
- `Cycle Time (วินาที)` หรือ `Cycle seconds`
- `ชิ้น/รอบ` หรือ `Units per cycle`
- `OEE (%)` หรือ `OEE`

### ข้อมูลงาน

- `งานผลิต`, `สินค้า`, `Product`, หรือ `Job`
- `รหัสงาน`, `Code`, หรือ `SKU`
- `ยอดผลิตต่อเดือน`, `Monthly demand`, หรือ `Demand`
- `กำหนดส่ง` หรือ `Due date`
- `ความสำคัญ` หรือ `Priority`

ถ้าไฟล์มีหลาย sheet แอปจะพยายามรวมข้อมูลจากทุก sheet มาวิเคราะห์ร่วมกัน

## การติดตั้งเป็นแอป

1. เปิดลิงก์ GitHub Pages
2. กดปุ่ม `ติดตั้งแอป` ถ้า browser รองรับ
3. บนมือถือสามารถใช้เมนู browser แล้วเลือก `Add to Home Screen`

## ไฟล์ในโปรเจกต์

- `index.html` โครงหน้าเว็บ
- `styles.css` หน้าตาและ responsive layout
- `app.js` สูตรคำนวณ การนำเข้าไฟล์ การสรุปผล และการบันทึกข้อมูล
- `manifest.webmanifest` ข้อมูลสำหรับติดตั้งเป็นแอป
- `service-worker.js` ระบบ cache สำหรับใช้งานออฟไลน์
- `icons/` ไอคอนแอป
